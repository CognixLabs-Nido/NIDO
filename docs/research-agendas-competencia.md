## 1. Resumen ejecutivo de cada app

### TYRA (tyra.io)

Origen sueco, presente en España. Enfoque "todo en uno" muy pulido para escuelas infantiles, con un modelo de datos claro y bien separado (Blog colectivo vs Álbum individual, Notas, Perfil). Fuerte en **gestión documental con firma digital** y en estructura para dirección. Cumple RGPD como argumento central. Login con PIN para tablets compartidas en el aula es un detalle inteligente.

### Schooltivity (schooltivity.com)

Producto español (Valencia, casualidad — coincide con tu zona). Muy enfocado al **mercado español**: cumple Veri\*factu, factura electrónica, fichaje digital de personal según ley vigente, modelo 233. Es el más completo en la parte **administrativa y de facturación**. Menos "moderno" visualmente pero muy práctico para un cole real en España.

### Mia Plus (mia.plus)

También español (Valencia). El más reciente y diferencial: ha integrado **IA (mIA)** para redacción de mensajes, rellenado de agendas, traducción a 100+ idiomas, resumen del día/semana, sugerencias de respuesta, búsqueda inteligente y alertas. Fuerte en **personalización por aula** y en pequeñas funciones de bienestar (desconexión digital, mensajes frecuentes, dictado a texto).

## 3. Patrones comunes (núcleo imprescindible de cualquier agenda Infantil)

Hay un **núcleo de funcionalidades que las tres comparten**, lo que indica que son el "mínimo viable" no negociable de este tipo de producto:

1. **Roles claros**: profe, familia, dirección/admin (tres permisos distintos).
2. **Modelo de datos**: Centro → Aulas/Grupos → Niños → Familiares vinculados al niño.
3. **Agenda diaria** con: comidas, sueño, deposiciones, observaciones libres.
4. **Comunicación bidireccional** profe-familia (mensajería + circulares).
5. **Galería multimedia** (fotos/vídeos) por niño y/o aula.
6. **Calendario** de eventos del centro.
7. **Ficha completa del niño** (alergias, contactos de emergencia, autorizados a recoger, médico).
8. **Asistencia** (entrada/salida y ausencias).
9. **Notificaciones push** (sin esto, la app no se usa).
10. **Cumplimiento RGPD** estricto (datos de menores).

## 4. Diferenciales que cada uno explota

- **Tyra**: claridad estructural (Blog vs Álbum vs Notas), firma digital, login PIN.
- **Schooltivity**: profundidad administrativa española (facturación, fichaje, modelo 233), informes pedagógicos, cuños virtuales.
- **Mia Plus**: IA aplicada (redacción, traducción, resumen), micro-eficiencias de UX (mensajes frecuentes, programación de envíos, dictado), bienestar del docente (desconexión digital).

---

## 5. Lo que ninguno hace bien (oportunidades para tu app)

- **Pictogramas / comunicación visual** para infantil con NEE o lengua extranjera.
- **Rutinas diarias visuales** que el niño pueda "seguir" en una pantalla del aula.
- **Integración con Google Classroom / Microsoft Teams** para coles que usan ya esos ecosistemas.
- **Modo offline real** para profes (rellenar agendas sin conexión y sincronizar).
- **Portal de familias accesible para abuelos / cuidadores** sin que tengan acceso completo a todo.
- **Exportación de portafolio del niño al final del curso** (PDF "libro del año").
- **Onboarding guiado** para profes mayores no nativas digitales.
- **Modo "varias profes en mismo aula"** sin pisarse al rellenar agenda.

## 6. Consideraciones legales (España, mayo 2026)

Si vas a manejar datos de menores y comunicaciones con familias en un cole español, ten en mente:

- **RGPD + LOPDGDD**: consentimientos explícitos para fotos, retención de datos, derecho al olvido, registro de actividades de tratamiento. Especialmente estricto con menores de 14.
- **Veri\*factu**: si vas a facturar a familias desde la app, software homologado obligatorio.
- **Registro horario**: si gestionas fichaje del personal, cumple ley vigente (registro inalterable).
- **Modelo 233**: declaración anual de gastos en guardería (las familias se deducen IRPF).
- **Hosting en UE**: para datos de menores, mucho más simple jurídicamente.

Para tu caso (un solo cole), la opción sensata es delegar Veri\*factu/fichaje a software de terceros y centrarte en lo pedagógico-comunicativo. **No reinventes lo que un Sage/Holded ya hace.**

## 7. Stack recomendado para tu caso

Dado tu perfil (programas, un solo cole, web + móvil) sigue siendo válido lo que comentamos:

- **Frontend**: Next.js 15 (App Router) con PWA habilitada. Una sola base de código para web y móvil instalable.
- **Backend / DB**: Supabase (auth, Postgres con RLS, storage para fotos, realtime para mensajería, Edge Functions para webhooks).
- **Notificaciones push**: web push API (gratis) + opcional OneSignal/Firebase si quieres más control.
- **UI**: Tailwind + shadcn/ui.
- **Hosting**: Vercel (frontend) + Supabase (backend).
- **Coste estimado**: ~0-50€/mes para un cole pequeño.
